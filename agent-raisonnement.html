01 Faire raisonner

CoT, ToT, ReAct

Trois patterns de raisonnement, trois formes différentes : une chaîne, un arbre, une boucle. Le troisième — ReAct — est le pont vers l'agentique.

2022 · Wei et al.

Chain-of-Thought

Le modèle déroule son raisonnement étape par étape avant de conclure. Visible, auditable, mais linéaire — si la première étape se trompe, tout dérape.
IDÉAL : problèmes logiques structurés, math, débogage step-by-step
2023 · Yao et al.

Tree-of-Thought

Plusieurs branches de raisonnement explorées en parallèle, évaluées, élaguées. Coûteux (N calls) mais bien plus robuste sur les problèmes ouverts.
IDÉAL : stratégie, créativité, problèmes à plusieurs solutions valides
2023 · Yao et al. · ICLR

ReAct

Le modèle alterne pensée et action : il pense, appelle un outil, observe, repense. C'est exactement ce que fait un agent moderne — ReAct est le squelette de toute boucle agentique.
IDÉAL : tâches dynamiques, recherche en contexte, tout ce qui nécessite des outils
Chain-of-Thought Tree-of-Thought ReAct
Forme Une chaîne linéaire Un arbre exploré Boucle pensée ↔ outil
Coût 1 call N calls (souvent 5–20) K calls (1 par tour)
Idéal pour Maths, logique, dette pas-à-pas Plans, stratégie, créativité Toute tâche avec outils ou monde extérieur
À éviter pour Tâches qui exigent des outils Quand le budget compte Problèmes purement clos
Maturité 2026 Production · partout Production sur cas premium Production · standard des agents

Nuance 2026 sur CoT. Les modèles à raisonnement natif (Claude 4.7, o-series, Gemini 3.x) raisonnent déjà en interne avant de répondre. Sur ces modèles, ajouter raisonne étape par étape dans le prompt a un effet faible voire négatif : ça force un raisonnement explicite en sortie, plus long, parfois moins bon que la chaîne interne. CoT reste utile sur les modèles non-raisonneurs et quand on veut auditer le raisonnement. Sur les autres, laissez le modèle raisonner seul.

Concrètement, ReAct dans un prompt

Pour cadrer un agent en ReAct, on n'a pas besoin de framework. Il suffit d'instruire le modèle à alterner les blocs <thought>, <action>, <observation> — le harnais intercepte chaque <action>, exécute l'outil, et injecte le résultat dans l'<observation> suivante.

✕ Avant — pensée libre
Aucune structure : l'agent invente sa réponse
Question : Quel est le coût mensuel
total des EC2 tagués env=prod et
tier=critical ?

// Réponse du modèle :
« D'après mes connaissances générales,
ça doit tourner autour de $12 000–$15 000
selon la taille de l'infra. »
Pourquoi ça rate : le modèle hallucine un chiffre. Aucun outil appelé. Aucun moyen de vérifier.
✓ Après — ReAct cadré
Pensée, action, observation explicites
<instructions>
Pour répondre, alterne :
<thought>...</thought>
<action>tool(args)</action>
<observation>...</observation>
Termine par <answer>.
</instructions>

// Trajectoire produite :
<thought>Lister les EC2 avec ces tags.</thought>
<action>aws_ec2.list(env=prod,
tier=critical)</action>
<observation>[42 instances...]</observation>
<thought>Sommer les prix mensuels.</thought>
<action>aws_pricing.sum(...)</action>
<observation>$8 472,30/mois</observation>
<answer>$8 472,30/mois (42 instances).</answer>
Pourquoi ça marche : chaque chiffre est traçable à un appel d'outil. Le raisonnement est auditable. Le harnais peut couper la boucle si une étape rate.

Le pattern ReAct n'est pas une option : c'est la grammaire d'un agent. Quand on parle de prompter un agent, on parle en grande partie de bien lui décrire comment alterner ses pensées et ses actions.

↗ Module Ce raisonnement « à voix haute » avant d'agir, c'est le mode thinking du modèle : Le LLM.
02 Patterns avancés

Quand un agent ne suffit pas

Quand un agent bien cadré ne suffit plus, certaines situations demandent un cran de plus : faire voter plusieurs réponses, faire critiquer l'agent par lui-même, ou — le plus puissant — séparer celui qui produit de celui qui juge.

Sample · vote

Self-consistency

On génère N réponses indépendantes (typiquement 5–10, température > 0), on garde la majoritaire. Élimine les coups de poisse sur les tâches à réponse unique.

IDÉAL : classification, extraction, math — tout ce qui a une réponse discrète.
Critique · itère

Self-Refine / Reflexion

L'agent critique sa propre sortie et itère sur K tours (souvent 2–3). Marche bien sur la rédaction, le refactoring, le résumé long. Limite : un modèle qui valide ses propres erreurs.

IDÉAL : écriture, refactoring, brouillon → version finale.
Anthropic · pattern dominant

Generator / Evaluator

Deux agents distincts, deux prompts distincts : l'un produit, l'autre note avec un système de scoring strict. Si le score est trop bas, l'output est rejeté ou repassé au Generator avec le verdict.

IDÉAL : production critique, génération de code, contenu visible utilisateur.
LLM optimise LLM

Meta-prompting

On utilise un modèle pour écrire ou améliorer un prompt. Anthropic et OpenAI proposent tous deux un prompt improver dans leur console. Très utile pour bootstrapper, mais ne remplace pas une vraie boucle d'éval (voir Eval-driven).

IDÉAL : bootstrapper un prompt depuis zéro, harmoniser un corpus de prompts existants.

Pourquoi Generator/Evaluator devient le standard

Il est bien plus facile de fabriquer un évaluateur impitoyable que d'apprendre à un générateur à être impitoyablement auto-critique.
— Recherche Anthropic, 2026

Le pattern marche parce qu'il découple deux compétences qu'un LLM ne maîtrise pas aussi bien dans la même tête : produire et juger. L'Evaluator a son propre prompt — souvent une rubrique de notation explicite — et une température zéro. C'est lui qui décide si l'output passe.

↗ Module Generator + Evaluator, c'est déjà deux agents qui se répartissent le travail : Multi-agent.
↗ Module Ces patterns, on les capitalise en briques réutilisables : Skills, hooks, rules.
03 À retenir

Raisonner, pour un agent, c'est boucler

Derrière chaque agent, il y a une boucle : penser, agir, observer, recommencer. Les patterns avancés ne font qu'ajouter un second regard sur cette boucle — celui qui juge.

Cinq points à emporter

1
Trois formes de raisonnement : la chaîne (CoT), l'arbre (ToT), la boucle (ReAct). On choisit selon le problème.
2
ReAct est la grammaire de l'agent : pensée → action → observation. Tout agent en est une variation.
3
Sur les modèles raisonneurs, ne force pas le « raisonne étape par étape » : ils le font déjà mieux en interne.
4
Pour fiabiliser : faire voter plusieurs réponses (self-consistency) ou itérer sur soi (self-refine).
5
Generator / Evaluator est le standard : un agent produit, un autre juge avec une grille stricte. Pas la même tête.
