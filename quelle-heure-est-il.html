Étude de cas · Mode outillé

« Quelle heure est-il ? »
Trois agents, une question.

Une question d'apparence triviale révèle, mieux que n'importe quelle théorie, ce que change un harnais. Voyons comment trois configurations différentes du même modèle de langage s'en sortent.

Pour chaque scénario, cliquez sur « Étape suivante » pour faire avancer la conversation pas à pas. Les annotations dorées en italique commentent ce qui se passe à l'intérieur du système.

L'utilisateur pose la même question dans les trois cas : « Quelle heure est-il ? » Le modèle, lui, est le même. Seul le harnais change.

01
LLM brut · pas de réflexion

Le réflexe : répondre quoi qu'il arrive

Aucun outil. Aucune étape de pensée. Le modèle reçoit la question et produit immédiatement la suite la plus probable.

— Conversation #1 —
Utilisateur
Quelle heure est-il ?
Modèle (réponse directe)
Il est 14 h 23.
Sauf qu'il est en réalité 09 h 47. Le modèle n'a aucune horloge interne — il vient de produire le texte qui ressemble le plus, statistiquement, à une réponse à cette question. C'est une hallucination typique : confiante, plausible, et complètement fausse.
✕
Échec silencieux. Le pire des cas : la réponse est fausse, mais l'utilisateur n'a aucun moyen de le savoir.

02
LLM avec réflexion · pas d'outils

L'introspection : reconnaître ses limites

Toujours aucun outil, mais le modèle a la possibilité de penser à haute voix avant de répondre.

— Conversation #2 —
Utilisateur
Quelle heure est-il ?
Réflexion interne
L'utilisateur me demande l'heure actuelle. Pour répondre, il me faudrait accéder à une horloge en temps réel. Or je suis un modèle de langage : je n'ai pas d'accès à l'état du monde au moment où on me parle. Je ne dispose d'aucun outil dans cette conversation. Je ne peux donc pas répondre. Je dois le dire honnêtement plutôt que d'inventer.
L'étape de réflexion permet au modèle de constater l'absence d'information fiable avant de s'engager dans une réponse. C'est une forme primitive d'auto-évaluation — utile, mais qui ne résout pas le problème de fond.
Modèle (réponse honnête)
Je n'ai pas accès à l'heure actuelle dans cette conversation. Je suis un modèle de langage sans accès à une horloge en temps réel. Si vous me dites l'heure, je pourrai en revanche vous aider à calculer un délai ou un fuseau horaire.
!
Honnête mais inutile. Le modèle évite l'erreur, mais l'utilisateur repart sans la moindre information. La question reste sans réponse.

Avant le scénario 3 · Ce qu'apporte le harnais

Passons maintenant au cas outillé. Voici ce que le harnais change.

Dans les deux premiers scénarios, le modèle était livré à lui-même : tout ce dont il disposait, c'était la question de l'utilisateur. Pour le troisième, on ajoute un harnais autour de lui — et la première chose que ce harnais fait, c'est doter le modèle d'outils qu'il pourra appeler en cas de besoin.

Encore faut-il que le modèle soit au courant de leur existence. La technique est simple, presque mécanique : le harnais réécrit silencieusement la requête avant de l'envoyer au modèle. La question de l'utilisateur reste là, mais elle est précédée du catalogue des outils disponibles — leur nom, leur description, leurs paramètres. Comme si l'on glissait une notice à côté d'une question avant de la transmettre.

Le modèle découvre alors, en plus de la question, quelque chose comme : « voici les outils que tu peux appeler, voici comment les invoquer, choisis-en un si tu en as besoin ». C'est ce qui va lui permettre de raisonner différemment des deux scénarios précédents : « tiens, il y a justement une fonction qui répond à cette question ».

Le scénario 3 commence donc par ce que le modèle reçoit vraiment — la question enrichie — avant d'enchaîner sur la boucle agentique elle-même.

03
LLM outillé · avec harnais boucle agentique

L'agent : penser, agir, observer, répondre

Le modèle dispose maintenant d'un outil get_current_time() fourni par son harnais. Suivons la mini-boucle qu'il déclenche.

— Conversation #3 · Boucle agentique —
Ce que l'utilisateur tape
Quelle heure est-il ?
Ce que le modèle reçoit réellement (prompt enrichi par le harnais)
// Instructions système
Tu es un assistant. Tu as accès aux outils listés ci-dessous. // Pour utiliser un outil, renvoie un appel au format JSON. // Le résultat te sera transmis avant de poursuivre.
// Outils disponibles
{ "name": "get_current_time", "description": "Renvoie l'heure actuelle dans un fuseau horaire donné.", "parameters": { "timezone": { "type": "string", "description": "ex. 'Europe/Paris', 'America/New_York'" } } }
// Message de l'utilisateur
Quelle heure est-il ?
Voilà ce qui arrive vraiment dans la « boîte aux lettres » du modèle. La question est en bas, mais elle est précédée du catalogue d'outils. C'est ce catalogue qui rend possible tout ce qui suit : sans lui, le modèle serait revenu au scénario 2.
Tour 1 · Pensée
On me demande l'heure. Je n'ai pas cette information par moi-même, mais je vois dans mes outils disponibles une fonction get_current_time qui renvoie l'heure actuelle. Je vais l'appeler.
Étape 1 — Pensée. Le modèle décide de ne pas inventer ; il choisit un outil.
Tour 1 · Action — appel d'outil
→ get_current_time
{
  "tool": "get_current_time",
  "arguments": { "timezone": "Europe/Paris" }
}
Étape 2 — Action. Le modèle ne peut pas exécuter l'outil lui-même : il décrit l'appel qu'il souhaite voir effectué, et il s'arrête là. C'est maintenant au harnais d'agir.
Tour 1 · Exécution — résultat de l'outil
← get_current_time
{
  "timestamp": "2026-04-29T09:47:12+02:00",
  "timezone": "Europe/Paris",
  "formatted": "mercredi 29 avril 2026, 09:47"
}
Étape 3 — Exécution. Le harnais a bel et bien lu l'horloge système et réinjecte le résultat dans la conversation, comme s'il s'agissait d'un nouveau message. Le modèle reprend la main.
Tour 2 · Pensée (observation)
J'ai reçu l'heure : 09:47 en heure de Paris, le 29 avril 2026. C'est ce que je cherchais. Je peux maintenant formuler une réponse claire pour l'utilisateur. La tâche est terminée, plus besoin d'appeler d'outil.
Étape 4 — Observation. Le modèle interprète le résultat et constate qu'il a tout ce qu'il faut. Il choisit de sortir de la boucle.
Réponse finale
Il est 9 h 47 à Paris (mercredi 29 avril 2026).
✓
Réponse correcte. Le même modèle qui hallucinait au scénario 1 et s'excusait au scénario 2 répond ici juste — parce que son harnais lui a donné les moyens d'aller chercher l'information.

Le modèle est le même. Le harnais fait toute la différence.

Dans les trois scénarios, c'est rigoureusement le même réseau de neurones qui calcule les réponses. Pourtant, l'expérience utilisateur va de « dangereusement faux » à « correctement répondu » en passant par « honnête mais stérile ».

Ce qui change, ce ne sont ni les paramètres du modèle ni son entraînement : c'est uniquement ce qui l'entoure. Un outil get_current_time, une boucle qui rappelle le modèle après l'exécution, un protocole pour passer le résultat. Quelques dizaines de lignes de code — et un agent émerge.
