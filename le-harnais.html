01 La métaphore

D'où vient ce mot étrange ?

À l'origine : le cheval

Un cheval, seul, est puissant mais ne sert à rien. Pour qu'il devienne utile, il faut un harnais : sangles, mors, rênes, attaches. Cet équipement ne change pas le cheval, mais le relie au monde et permet de le diriger.

Puissance brute → Puissance utile

En IA : le modèle

Un grand modèle de langage (LLM), seul, ne fait qu'une chose : produire du texte. Pour qu'il agisse — lire des fichiers, naviguer, exécuter du code — il lui faut tout un équipement autour. Cet équipement, c'est le harnais.

Texte brut → Action dans le monde
02 Définition

Qu'est-ce qu'un harnais, précisément ?

Le harnais (« harness » en anglais) est l'ensemble du code, des outils, des règles et des boucles d'exécution qui entoure un modèle d'IA pour le transformer en agent capable d'agir de façon autonome dans un environnement.

Le modèle

Le « cerveau » : il décide quoi faire à chaque étape.

Le harnais

Le « corps » : tout ce qui exécute, observe et contrôle.

L'environnement

Le « monde » : fichiers, web, APIs, systèmes que l'agent manipule.

03 Pourquoi en a-t-on besoin ?

Un LLM, tout seul, ne peut pas grand-chose

Imaginez ChatGPT à l'état brut : une fonction qui prend du texte en entrée et renvoie du texte en sortie. C'est tout. Sans harnais, voici ce qui lui manque.

Aucun accès aux fichiers

Ne peut ni lire votre code, ni ouvrir un PDF, ni écrire un document.

Aucun accès au web

Ne peut pas chercher d'info actuelle, ni consulter une page, ni appeler une API.

Aucune mémoire persistante

Oublie tout entre deux conversations. Pas de notes, pas d'historique propre.

Aucune boucle d'exécution

Ne peut pas dire « je viens d'essayer X, ça a échoué, j'essaie Y ».

Aucune limite ni garde-fou

Sans cadre, rien n'empêche les actions dangereuses ou les boucles infinies.

Aucune perception du résultat

Produit du texte mais ne « voit » jamais l'effet réel de ce qu'il propose.

04 Le fonctionnement

Le cœur du harnais : la boucle agentique

Le harnais orchestre un cycle qui se répète tour après tour, jusqu'à ce que la tâche soit terminée. À chaque tour, le modèle décide, le harnais exécute, et le résultat redevient une nouvelle entrée.

05 Anatomie d'un harnais

Ce qu'il y a, en pratique, dans un harnais

Les outils (tools)

Fonctions que le modèle peut appeler : lecture/écriture de fichiers, recherche web, exécution de code, requêtes SQL, envoi d'emails, etc.

La boucle d'exécution

Le code qui rappelle le modèle à chaque tour, exécute les actions, transmet les résultats. C'est ce qui permet l'autonomie.

La gestion du contexte

Décide de ce qu'on garde, résume, ou évacue de la conversation pour ne pas dépasser la fenêtre du modèle.

Les garde-fous

Limites de temps, d'argent, d'actions ; vérifications avant exécution ; permissions ; sandbox d'exécution.

L'observation & la trace

Logs détaillés de chaque étape pour pouvoir comprendre, déboguer ou auditer le comportement de l'agent.

Le critère d'arrêt

Comment l'agent sait que la tâche est finie : signal explicite du modèle, succès d'un test, limite atteinte…

06 Un exemple concret

Un agent qui code

« Corrige le bug dans mon application Python. » Voici comment un harnais d'agent de codage transforme cette demande en travail réel.

1

Lire

Liste les fichiers, ouvre ceux qui semblent pertinents, comprend la structure.

2

Analyser

Identifie l'origine probable du bug, formule une hypothèse précise.

3

Modifier

Édite les fichiers concernés via l'outil d'édition du harnais.

4

Tester

Lance la suite de tests dans une sandbox isolée, observe la sortie.

5

Itérer

Si un test échoue, ajuste et recommence. Sinon, signale la fin.

À chaque phase, c'est le harnais qui rend l'action possible : il fournit l'outil, l'exécute, capture la sortie, et la renvoie au modèle. Le modèle, lui, ne fait que décider.

07 Qui fait quoi ?

Modèle vs harnais : la division du travail

Le modèle · décide

  • Comprendre la demande
  • Choisir une stratégie
  • Décider quel outil utiliser
  • Formuler les bons paramètres
  • Interpréter ce qu'il observe
  • Reconnaître quand s'arrêter

Le harnais · exécute

  • Mettre les outils à disposition
  • Exécuter chaque action demandée
  • Renvoyer les résultats au modèle
  • Gérer la mémoire et le contexte
  • Faire respecter les garde-fous
  • Tracer, logger, surveiller

« Le modèle est l'intelligence ; le harnais est la possibilité d'agir. Ensemble, ils forment l'agent. »

08 Pourquoi c'est crucial

Le harnais, c'est souvent là que tout se joue

Avec le même modèle, deux harnais différents peuvent produire des agents radicalement différents — l'un brillant, l'autre inutilisable.

09 Le spectre des harnais

Tous les harnais ne se ressemblent pas

Du plus minimal au plus complet, on rencontre une gradation continue. Plus on monte, plus l'agent peut faire de choses sans aide humaine — et plus il faut soigner les garde-fous.

2
3
4

Minimal

Chatbot

Aucun outil. Le modèle ne fait que produire du texte. Pas vraiment un « agent ».

Outillé

Assistant

Quelques outils ponctuels (recherche web, calcul). L'humain garde la main à chaque étape.

Agentique

Agent autonome

Boucle complète : le modèle agit en plusieurs tours sans intervention pour des tâches définies.

Multi-agent

Système d'agents

Plusieurs agents coordonnés, chacun avec son harnais, qui se passent des sous-tâches.

10 Les défis du concepteur

Concevoir un harnais : un art difficile

Trois tensions reviennent dans toutes les conceptions de harnais. Bien les arbitrer, c'est faire la différence entre un agent jouet et un agent fiable.

Autonomie↔Sécurité

Plus on laisse l'agent libre d'agir, plus il accomplit de choses — mais plus il peut aussi causer de dégâts. Le harnais doit ouvrir la porte juste assez.

Contexte↔Coût

Donner tout l'historique au modèle améliore ses décisions, mais coûte cher en tokens. Le harnais doit choisir intelligemment ce qu'il garde.

Richesse↔Simplicité

Beaucoup d'outils donnent à l'agent plus de pouvoir, mais le saturent et le perdent. Un bon harnais offre les bons outils, pas tous les outils.

12 À retenir

Si vous ne deviez retenir que cinq choses…

1

Un LLM seul ne fait que produire du texte. Le harnais est ce qui le rend capable d'agir.

2

Le harnais comprend des outils, une boucle d'exécution, une gestion du contexte et des garde-fous.

3

À chaque tour, le modèle décide ; le harnais exécute, observe, et renvoie le résultat.

4

Avec le même modèle, deux harnais peuvent produire des agents radicalement différents.

5

Concevoir un bon harnais, c'est arbitrer entre autonomie, sécurité, contexte, coût et richesse d'outillage.

13 Pour aller plus loin

Et maintenant ?

Trois pistes concrètes pour passer de la compréhension théorique à la pratique.

01

Observer un harnais en action

Lancer un agent comme Claude Code ou un agent open-source, regarder ses logs, suivre ce qu'il appelle à chaque tour.

02

Lire un harnais existant

Plusieurs frameworks open-source publient leur code : c'est la meilleure école pour comprendre les choix concrets.

03

Construire un mini-harnais

Une boucle de quelques dizaines de lignes avec deux ou trois outils suffit pour saisir profondément le concept.
